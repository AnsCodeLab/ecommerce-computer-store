# E-Commerce Computer Store

A Flask + SQLite e-commerce computer store: REST API + a production-style server-rendered web storefront with a session cart and checkout.

The storefront has a modern UI — sticky navbar with a live cart badge, hero banner, responsive
product-card grid with in-stock/low/out badges, item detail pages, a cart table, a two-column
checkout with order summary, and an order-confirmation panel. Styling is a self-contained
stylesheet (`app/static/css/style.css`, no external CDNs — works offline on a LAN).

**Built autonomously via [opencode](https://opencode.ai) + the [superpowers](https://github.com/AnsCodeLab/superpowers) skills (lean-16k fork) driving a local `Qwen3-8B` model on `llama.cpp`** — following the full flow: brainstorm → design → plan → TDD → implement → review.

## Features

- **Catalog:** categories and items (products) with price, stock, and category.
- **Customers:** create / list / get / delete.
- **Orders & checkout:** validates stock, creates an order with line items, decrements stock, computes totals.
- **Web storefront:** browse items by category, item detail pages, session-based cart, and checkout that places a real order.
- **SQLite** storage, **pytest** test suite (30 tests), TDD throughout.

## Layout

```
app/
  __init__.py     # app factory create_app(db_path), get_db(), blueprint registration
  db.py           # schema, get_conn, init_db, seed
  categories.py   # /api/categories  CRUD
  items.py        # /api/items       CRUD
  customers.py    # /api/customers   CRUD
  orders.py       # /api/checkout, /api/orders
  web.py          # /, /item/<id>, /cart, /cart/add, /cart/remove, /checkout
  templates/      # base, index, item_detail, cart, checkout, confirmation (Jinja2)
  static/css/     # style.css (self-contained theme)
tests/            # pytest, one file per module (Flask test client + temp DB)
docs/superpowers/ # design spec + implementation plan (generated by the local model)
docs/CODE-REVIEW.md
run.py            # seeded entrypoint on http://127.0.0.1:5005
```

## Run

```bash
python3 -m venv .venv && . .venv/bin/activate
pip install -r requirements.txt
python run.py          # http://127.0.0.1:5005  (DB seeded with sample data)
```

Expose on your LAN (reachable from other devices at `http://<your-ip>:5005`):

```bash
HOST=0.0.0.0 python run.py     # HOST/PORT/DB_PATH are env-configurable
```

## Test

```bash
pytest -q              # 30 passed
```

## REST API

| Method | Path | Body | Notes |
|--------|------|------|-------|
| GET/POST | `/api/categories` | `{name}` | list / create (201) |
| GET/DELETE | `/api/categories/<id>` | — | get (404 if missing) / delete (204) |
| GET/POST | `/api/items` | `{name,description,price,stock,category_id}` | `description`/`category_id` optional |
| GET/DELETE | `/api/items/<id>` | — | |
| GET/POST | `/api/customers` | `{name,email}` | |
| GET/DELETE | `/api/customers/<id>` | — | |
| POST | `/api/checkout` | `{customer_id, items:[{item_id,quantity}]}` | validates stock, returns order + total (201) |
| GET | `/api/orders`, `/api/orders/<id>` | — | order with line items + total |

## How it was built

The local Qwen3-8B generated the design spec, plan, tests, and implementation per module; a controller (Claude) ran pytest after each phase and fixed the model's mistakes (the review/fix step of the superpowers flow). See `docs/superpowers/` for the generated spec and plan, and `docs/CODE-REVIEW.md` for the model's self-review.
